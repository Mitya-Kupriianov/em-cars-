# Предзагрузка контента наперёд + фиксация размеров секций

Дата: 2026-06-17
Статус: одобрено к реализации

## Проблема

При прокрутке сайта пользователь замечает две вещи:

1. **Лаг картинок.** `CarImage` использует `next/image` с нативным lazy-loading, который
   стартует загрузку, только когда картинка почти в зоне видимости. Человек видит
   момент подгрузки (пустое место / placeholder → картинка).
2. **Дёрганье layout (CLS).** Секции на главной (`popular cars`, `offices`, ранее
   `banners`) грузятся через `fetch` в `useEffect`. До прихода данных секция схлопнута
   в ноль высоты; когда контент появляется — всё ниже «прыгает».

## Цель

- Грузить контент **наперёд, примерно на один экран вперёд** («150%»: видимый экран +
  ~половина-целый экран буфера), чтобы к моменту появления в кадре контент уже готов.
- **Зарезервировать размеры всех тяжёлых секций**, чтобы ничего не сдвигалось при
  загрузке.
- Применить **единообразно на всех основных страницах** (главная, каталог, страница
  авто, блог).

## Ограничения и факты по стеку (проверено)

- Next.js 16.2.7. В Next 16 проп `priority` у `next/image` **устарел** (вместо него
  `preload`). `next/image` **не даёт** настраивать зону упреждения (`rootMargin`) у
  нативного lazy-loading — только `loading="lazy" | "eager"`. Поэтому «1 экран вперёд»
  реализуем своим `IntersectionObserver`.
- Картинки авто в `CarCard` уже обёрнуты контейнером с `aspect-[16/10]` + `<Image fill>`,
  т.е. место под картинку зарезервировано — проблема только в *моменте* старта загрузки.
- В каталоге (`catalog/page.tsx`) скелетоны и `min-height` контейнера списка уже есть.
  Авто-подгрузку партий вместо кнопки «Загрузить ещё» НЕ делаем (вне запроса).

## Решение (вариант A)

Два переиспользуемых примитива + точечное применение.

### 1. Примитив предзагрузки картинок

Новый хук `useNearViewport(ref)`:
- На основе `IntersectionObserver` с адаптивной зоной упреждения (`rootMargin`):
  - десктоп (`min-width: 768px`): `"100% 0px"` (≈ один экран вперёд);
  - мобайл: `"50% 0px"` (≈ половина экрана — бережём мобильный трафик).
- Срабатывает один раз: как только элемент вошёл в расширенную зону — возвращает `true`
  и отписывается (observer disconnect).
- Безопасен при SSR (observer создаётся в `useEffect`); при отсутствии
  `IntersectionObserver` (старые окружения) — фолбэк `true` (грузим сразу).

Доработка `CarImage`:
- Добавляется опциональный режим предзагрузки наперёд (например, проп `preloadAhead`,
  по умолчанию включён для карточек/галереи).
- Контейнер по-прежнему держит размер (`aspect-ratio` родителя + `fill`).
- `<Image>` получает `loading: near ? "eager" : "lazy"`, где `near` приходит из
  `useNearViewport`. До входа в зону — `lazy` (резерв места держит контейнер), после —
  переключение на `eager` форсирует старт загрузки за ~экран до появления.

Итог: картинка догружается **до** появления в кадре, место занято заранее → нет ни
лага, ни прыжка.

### 2. Применение к картинкам

- `CarCard` → каталог, главная (popular), избранное, сравнение (один компонент —
  правка в одном месте).
- Галерея на странице авто (`catalog/[slug]/CarDetailClient.tsx`).
- Блог (`blog/page.tsx`, `blog/[slug]/ArticleClient.tsx`).
- Сырые `<img>` логотипов в `Header`/`Footer` — наоборот, явный eager (они вверху,
  должны грузиться сразу). `VideoReviews` уже `loading="lazy"` — не трогаем.

### 3. Фиксация секций (анти-прыжок)

- **Главная → popular cars:** пока `popularCars.length === 0`, рендерим 3 skeleton-
  карточки (повторно используем разметку скелетона из каталога), чтобы grid держал
  высоту. Сейчас при пустом массиве секция схлопнута и контент ниже прыгает при приходе
  данных.
- **Главная → offices:** контейнеру списка офисов задаём `min-height`, чтобы замена
  дефолтных офисов на серверные не двигала layout.
- **Длинные статичные секции** (About/Advantages, WhyUs, VideoReviews, Footer):
  утилитный класс `content-visibility: auto; contain-intrinsic-size: auto <H>` (через
  globals.css). Браузер резервирует примерную высоту и пропускает рендер вне экрана —
  дёшево, ускоряет скролл, без JS.

### 4. Новые/изменяемые единицы

| Единица | Что делает | Зависимости |
|---|---|---|
| `useNearViewport` (новый хук) | Сообщает, вошёл ли элемент в зону упреждения ~1 экран | `IntersectionObserver` |
| `CarImage` (правка) | Предзагрузка наперёд через `useNearViewport`, переключение `loading` | `useNearViewport`, `next/image` |
| `CarCardSkeleton` (новый/вынесенный) | Скелетон карточки для резерва высоты | — |
| Главная `page.tsx` (правка) | Скелетоны popular + `min-height` offices | `CarCardSkeleton` |
| `globals.css` (правка) | Класс `.cv-section` для `content-visibility` | — |

## Проверка (definition of done)

Запустить dev-preview, проскроллить главную и каталог и подтвердить:
1. В Network загрузка картинок стартует за ~экран до их появления в кадре (не в момент
   входа в viewport).
2. Нет видимого layout shift при загрузке секций главной (скелетоны держат высоту до
   прихода данных).
3. Скролл плавный; ничего не «прыгает» на десктопе и мобиле.

## Вне scope (YAGNI)

- Авто-подгрузка партий (infinite scroll) в каталоге вместо кнопки «Загрузить ещё».
- Виртуализация списков.
- Полный lockdown CSP / прочие несвязанные изменения.
